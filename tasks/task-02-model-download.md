# Задача 2. Подготовить загрузку моделей

## Цель

Подготовить воспроизводимый способ локального скачивания моделей, которые будут использоваться chat- и embedding-сервисами.

## Что нужно сделать

Нужно добавить в репозиторий скрипт:

- `download_models.sh`

Скрипт должен:

- создавать каталог `models/`, если его нет;
- скачивать chat-модель;
- скачивать embedding-модель;
- сохранять их в понятные локальные пути.

На этом этапе используются зафиксированные модели:

- `Qwen/Qwen3-1.7B-Instruct-GGUF`
- `Qwen/Qwen3-Embedding-0.6B-GGUF`

Нужно ориентироваться на заранее выбранный формат GGUF и разумное квантование для CPU-only сценария.

## Ограничения

- Не добавлять сложные менеджеры зависимостей
- Не внедрять дополнительную логику, не связанную со скачиванием моделей
- Не менять выбранные модели без отдельного согласования

## Ожидаемый результат

В репозитории есть простой и понятный скрипт, который скачивает обе модели в локальный каталог `models/`.

## Проверка результата

Результат считается достигнутым, если:

1. В репозитории присутствует `download_models.sh`
2. Скрипт выполняется без ошибок
3. После выполнения обе модели лежат в `models/`
4. Имена и пути моделей понятны и пригодны для дальнейшего использования в `docker-compose.yml`
