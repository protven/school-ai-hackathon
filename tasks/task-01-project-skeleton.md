# Задача 1. Подготовить базовый каркас локального проекта

## Цель

Подготовить минимальный инфраструктурный каркас проекта, достаточный для дальнейшего подключения моделей и запуска сервисов.

## Что нужно сделать

Нужно добавить в репозиторий базовые файлы и каталоги, которые будут использоваться в следующих задачах:

- `docker-compose.yml`
- `.env.example`
- каталог `models/`
- каталог `data/`

Также нужно заранее зафиксировать имена сервисов в `docker-compose.yml`:

- `open-webui`
- `llama-chat`
- `llama-embed`

На этом этапе не требуется добиваться полностью рабочего запуска RAG.  
Нужно только подготовить корректный каркас проекта.

## Ограничения

- Не добавлять лишние сервисы
- Не использовать `vLLM`, `LiteLLM`, `Postgres`, `Redis`, `Nginx`
- Не усложнять compose-файл лишними сущностями
- Не выходить за рамки MVP

## Ожидаемый результат

В репозитории есть минимальная инфраструктурная основа для следующих шагов реализации.

## Проверка результата

Результат считается достигнутым, если:

1. В репозитории присутствуют `docker-compose.yml` и `.env.example`
2. В репозитории есть каталоги `models/` и `data/`
3. В compose-файле зафиксированы сервисы:
   - `open-webui`
   - `llama-chat`
   - `llama-embed`
4. Команда `docker compose config` выполняется без ошибок
