# Промпт для подготовки списка задач агента-кодера

Нужно на основе готового технического проекта подготовить короткий список задач для агента-кодера.

## Контекст

Есть технический проект по учебному локальному RAG-сервису для школьного хакатона.

Проект уже зафиксирован на уровне архитектуры, стека, моделей, ограничений и MVP.

Теперь нужно не писать большую постановку на всю реализацию сразу, а разложить работу на небольшой список отдельных задач.

## Задача

Подготовить документ `docs/coder-agent-tasks.md` со списком задач для агента-кодера.

## Требования к задачам

Нужно составить от 3 до 5 задач.

Каждая задача должна быть:

- атомарной;
- проверяемой;
- ограниченной по объему;
- логически завершенной;
- укладывающейся в границы MVP.

## Что должно быть у каждой задачи

Для каждой задачи нужно указать:

- название;
- что нужно сделать;
- ожидаемый результат;
- как проверить результат.

## Ограничения

- Не смешивать в одной задаче несколько крупных целей.
- Не выходить за рамки MVP.
- Не добавлять лишние сервисы и необязательную инфраструктуру.
- Не писать подробную реализационную инструкцию — только список задач.

## Ожидаемый результат

Нужен короткий и понятный список задач, который станет основой для следующих отдельных постановок агенту-кодеру.

## Дополнительное требование

Список задач должен отражать естественный порядок реализации проекта.

Ожидаемый характер задач:
- подготовка каркаса проекта;
- подготовка моделей;
- запуск локального чата;
- подключение embeddings и проверка RAG.

Но итоговый список должен формироваться строго на основе технического проекта.
