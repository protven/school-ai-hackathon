# Промпт для подготовки технического проекта

Нужно на основе готового исследования подготовить технический проект для репозитория.

## Контекст

Есть исследовательский документ по учебному проекту для школьного хакатона.

Проект: простой локальный RAG-сервис, который запускается на обычном ноутбуке без GPU.

Пользователь загружает несколько небольших документов и задает по ним вопросы в чате. Система находит релевантные фрагменты текста и формирует ответ с учетом найденного контекста.

Исследование уже выполнено и сохранено в `research/deep-research.md`.

## Задача

Нужно преобразовать исследовательский документ в компактный и практический технический проект.

## Что должно быть в техническом проекте

Документ должен зафиксировать:

- цель проекта;
- пользовательский сценарий;
- выбранный стек;
- выбранные модели;
- общую архитектуру;
- ограничения;
- границы MVP;
- критерии готовности.

## Что не нужно делать

- Не нужно пересказывать все исследование.
- Не нужно оставлять альтернативы, если уже есть обоснованный выбор.
- Не нужно превращать документ в постановку на реализацию.
- Не нужно делать документ слишком длинным.
- Не нужно добавлять лишнюю инфраструктуру вне MVP.

## Требования к результату

Технический проект должен быть:

- коротким;
- понятным;
- практическим;
- пригодным для использования в репозитории;
- достаточным как основа для дальнейшей декомпозиции задач.

## Формат результата

Нужен файл `docs/technical-project.md`.

Желательная структура:

1. Идея проекта
2. Цель
3. Пользовательский сценарий
4. Выбранный стек
5. Выбранные модели
6. Общая архитектура
7. Ограничения
8. Границы MVP
9. Критерии готовности

## Дополнительное требование

Документ должен отражать уже принятые решения и служить основой для следующего шага — составления списка задач для агента-кодера.
