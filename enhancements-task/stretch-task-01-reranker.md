# Задача со звездочкой 1. Добавить reranker

## Цель

Улучшить качество RAG-поиска за счет дополнительного ранжирования найденных фрагментов после первичного retrieval.

## Что нужно сделать

- исследовать, какой минимальный reranker можно добавить без сильного усложнения проекта;
- определить, будет ли reranking встроен в текущий поток Open WebUI или вынесен в отдельный сервис/обработчик;
- подключить reranker только если это не ломает базовый сценарий и не делает проект слишком тяжелым для учебного использования.

## Ожидаемый результат

На части запросов ответы по документам становятся более точными за счет лучшего выбора релевантных фрагментов.

## Как проверить

- есть тестовый набор вопросов по документу;
- без reranker и с reranker можно сравнить качество retrieval;
- проект остается воспроизводимым и не выходит за рамки разумной сложности.
